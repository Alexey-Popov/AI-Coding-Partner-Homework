# Homework 4: 4-Agent Pipeline

## Overview

This project implements a **4-agent pipeline** for automated bug fixing, security verification, and unit test generation. The pipeline operates on a real Express.js API bug (API-404: GET /api/users/:id returns 404 for valid user IDs).

## Pipeline Architecture

```
Bug Researcher → Research Verifier → Bug Planner → Bug Implementer
                                                         │
                                              ┌──────────┴──────────┐
                                              ▼                     ▼
                                    Security Verifier    Unit Test Generator
```

**Run order**:
1. Bug Researcher (manual/human step — produces `codebase-research.md`)
2. **Bug Research Verifier** — verifies research accuracy; produces `verified-research.md`
3. Bug Planner (manual/human step — produces `implementation-plan.md`)
4. **Bug Implementer** — applies fixes; produces `fix-summary.md`
5. **Security Verifier** — reviews changed code; produces `security-report.md`
6. **Unit Test Generator** — generates and runs tests; produces `test-report.md` + test files

## The Bug

**Bug ID**: API-404
**File**: `demo-bug-fix/src/controllers/userController.js:23`
**Root cause**: `req.params.id` is a string, but users array has numeric IDs. Strict equality `===` always fails → 404 for every user lookup.
**Fix**: `users.find(u => u.id === parseInt(userId, 10))`

## Agents

### Task 1: Bug Research Verifier
**File**: `agents/research-verifier.agent.md`

Verifies every file:line reference in `codebase-research.md` against the actual source. Applies the Research Quality Measurement skill to assign a quality level (EXCELLENT/GOOD/ACCEPTABLE/POOR/INSUFFICIENT) and produces `verified-research.md`.

### Task 2: Bug Implementer
**File**: `agents/bug-implementer.agent.md`

Reads `implementation-plan.md`, applies each code change exactly as specified, runs tests after every change, and stops on failure. Produces `fix-summary.md` with before/after code, test results, and manual verification steps.

### Task 3: Security Vulnerabilities Verifier
**File**: `agents/security-verifier.agent.md`

Reads `fix-summary.md` and all changed files. Scans for injection, hardcoded secrets, insecure comparisons, missing validation, unsafe deps, XSS/CSRF. Rates findings as CRITICAL/HIGH/MEDIUM/LOW/INFO. Produces `security-report.md` — no code edits.

### Task 4: Unit Test Generator
**File**: `agents/unit-test-generator.agent.md`

Reads `fix-summary.md` and changed files. Generates tests for changed code only, following FIRST principles. Runs tests and records results in `test-report.md`.

## Skills

### Research Quality Measurement (`skills/research-quality-measurement.md`)
Defines quality levels for bug research: EXCELLENT / GOOD / ACCEPTABLE / POOR / INSUFFICIENT with scoring criteria (reference accuracy, snippet fidelity, root cause correctness, completeness, clarity).

### FIRST Unit Tests (`skills/unit-tests-FIRST.md`)
Defines FIRST test principles: Fast, Independent, Repeatable, Self-validating, Timely. Used by the Unit Test Generator agent to ensure test quality.
